# Pocket Coding — 页面功能设计 & Review

> 配套可视化稿：`mockups/index.html`（5 个视觉版本，每个版本画出全部核心页面）。
> 设计系统（来自 ui-ux-pro-max）：暗色 OLED、开发者工具配色（slate + 成功绿 `#22C55E`）、字体 Inter + JetBrains Mono。

移动端设计哲学：**不是把 IDE 搬到手机，而是「AI Agent 指挥官 + 结果审阅台」**。主体验 = 对话 / 审阅 / 审批，输入尽量靠语音与快捷指令。

---

## 核心页面清单（7 个必画页面）
| # | 页面 | 角色 | 出现底部导航 |
|---|---|---|---|
| P1 | 连接 / 配对 Connect | 冷启动，绑定云主机 | 否 |
| P2 | 首页 Home | 项目 / 会话枢纽 | 是 |
| P3 | 会话 Chat（核心） | 对话式编程主战场 | 是 |

## P1 · 连接 / 配对 Connect
**功能**
- 输入云主机地址（IP / 域名）+ 一次性**配对码**（扫码或手输）完成设备配对。
- 连接后**自动探测已安装工具**（claude / codex / cursor-agent），以可启用 chip 展示版本。
- 连接状态可视化（测试连接、TLS 标识）。

**关键交互**：扫码优先；连接中骨架态；失败给明确恢复路径（重试/查看文档）。

**Review**
- ✅ 遵循 `primary-action`：整屏只有一个主 CTA「连接」。
- ✅ `input-type-keyboard`：地址用 url 键盘、配对码用数字键盘。
- ⚠ 安全提示：需明示「配对码仅一次有效 / 全程 TLS」，降低公网主机的信任焦虑。
- ⚠ `empty-nav-state`：工具未探测到时要解释原因（未安装/未在 PATH），而非静默隐藏。

## P2 · 首页 Home
**功能**
- 顶部**主机切换器**（多云主机，显示在线状态）。
- 「项目/工作区」列表：名称 + git 分支 + 状态（clean / +N 改动）。
- 「最近会话」列表：工具徽标 + 最后一句 + 状态点（运行中/等审批/完成）。
- FAB「新建会话」→ 选择 项目 / 工具 / 模型 / 模式。

**关键交互**：会话卡片状态点用颜色**并配文字/图标**（不靠颜色单一表意）；下拉刷新。

**Review**
- ✅ `nav-hierarchy`：底部 5 项主导航（Home/Chat/Files/Terminal/Settings），主机切换属次级放顶部。
- ✅ `bottom-nav-limit`：正好 5 项，图标+文字。
- ⚠ 会话多时需 `virtualize-lists`（50+ 虚拟滚动）。
- ⚠ 运行中会话应有轻动效呼吸点，强化「云端在跑」的感知。

## P3 · 会话 Chat（核心）
**功能**
- 头部：项目名 + 当前工具徽标（Claude Code）+ 模型 + **运行状态条**（思考中/执行中/等待审批）。
- **卡片化消息流**（区别于普通聊天）：用户消息、AI 文本(Markdown)、**Plan 卡片**、**Tool Call 卡片**（可展开、危险命令高亮）、**Diff 卡片**（+X -Y，点进 P4）、命令输出卡片、状态卡片。
- 输入区：**语音**（长按说话转写）、文本、`@` 引用文件、快捷指令 chips（/plan /fix /test）、工具&模型切换弹层、**停止**按钮（中断 Agent）。

- **检查点标记**：消息流中每个「用户消息→改动」回合前插入一条轻量**检查点标记**（`◇ 检查点 · 改动前`），点击进 P7；顶部提供「预览」「历史」入口。

**关键交互**：卡片**左滑 Approve / 右滑 Reject**；代码块一键复制；Agent 执行中随时打断；点检查点标记可回退。

**Review**
- ✅ `swipe-clarity` + `gesture-alternative`：滑动审批要有明显 affordance，且同时提供可见的 ✔/✗ 按钮兜底。
- ✅ `destructive-emphasis`：危险命令（rm/push）审批用红色语义、与主操作分离。
- ✅ `progressive-loading`：Agent 思考 >1s 用骨架/闪烁，而非空白。
- ⚠ `touch-target-size`：输入区麦克风/发送 ≥44pt；底部栏避开手势条（安全区）。
- ⚠ 长会话需锚定「跳到最新」按钮 + 保留滚动位置（`state-preservation`）。
- ⚠ 检查点标记要低调（次要视觉层级），不打断对话阅读流。

## P4 · Diff 审阅
**功能**
- 顶部文件 tab（多文件切换）+ 改动摘要（+X -Y）；diff **始终以当前 baseline 为基准**。
- 分 hunk 展示，逐块 ✔/✗；支持「全部接受」/「按文件接受」。
- 底部双动作：**接受变更**（→ baseline 前移、prune 本次改动前快照、只保留对话）/ **回退到改动前**（restore 检查点，丢弃本次改动）。

**关键交互**：等宽字体 + 语法高亮；双指缩放看代码；行号 tabular 对齐；接受后该 diff 归零、baseline 更新。

**Review**
- ✅ `number-tabular`：行号/增删数用等宽数字，避免抖动。
- ✅ `color-not-only`：增删除颜色外，用 `+ / -` 符号与背景区分（色盲友好）。
- ✅ `confirmation-dialogs` + `undo-support`：接受/回退前确认；接受为不可逆里程碑要**二次确认**并说明「快照将被清理」。
- ✅ `destructive-emphasis`：「回退到改动前」属破坏性操作，与「接受」在视觉上分离、用警示色。
- ⚠ 移动端 hunk 过宽 → 横向可滚但主纵向滚动不被打断（`scroll-behavior`）。

## P5 · 终端 Terminal
**功能**
- 真·PTY 直连终端（xterm 风格），查看完整日志、手动执行命令。
- 顶部标识主机；输出区支持复制。
- 软键盘上方**快捷键条**：Esc / Tab / Ctrl / 方向键（终端必备但手机缺失的键）。

**Review**
- ✅ `readable-font-size` + 等宽字体；`safe-area-awareness` 快捷键条避开手势条。
- ✅ `standard-gestures`：纵向滚动看历史，不劫持系统返回手势。
- ⚠ 移动端终端是兜底能力，入口不宜过于突出，避免喧宾夺主。

## P6 · Web 预览 Preview（前后端联调 · 需求①）
**功能**
- 顶部 **URL 栏** + **设备宽度切换**（Mobile / Tablet / Desktop）+ 刷新 / 外部浏览器打开。
- 主体 **内嵌 WebView**，经「带鉴权反向代理」访问云主机上正在跑的 dev server；前端直连同主机后端 API，实现**前后端联调**。
- 底部工具条：**启动/重启 dev server**、**运行日志 / 控制台**、**网络请求**面板。
- 特色：**「选取元素 → 发送到对话」**——点选页面节点，把选择器/截图回传给 Agent 说「改这里」。

**关键交互**：dev server 未启动时给明确 CTA「启动预览」；启动中显示端口探测/编译进度；HMR 热更新实时生效。

**Review**
- ✅ `primary-action`：未运行态整屏聚焦一个「启动预览」CTA。
- ✅ `progressive-loading`：编译/端口探测用进度态，避免白屏焦虑。
- ✅ `input-type-keyboard`：URL 栏用 url 键盘。
- ⚠ 安全提示：预览经短时令牌鉴权、dev server 不直接对公网，需在 UI 弱标识「安全代理」。
- ⚠ WebView 需处理软键盘遮挡与安全区；横向内容可缩放（`scroll-behavior`）。
- ⚠ 选取元素模式要有明显进入/退出态，避免误触。

## P7 · 检查点时间线 Checkpoints（对话历史 + 回滚 · 需求②）
**功能**
- 纵向**时间线**，每节点 = 一次「用户消息 → Agent 改动」回合：摘要、改动文件（+X -Y）、状态（**待接受 / 已接受 / 已回退**）。
- **待接受**节点可「**回退到本次改动前**」（restore 快照，丢弃改动）。
- **已接受**节点显示为 **baseline 里程碑**（快照已清理、仅留对话，不可回退到其之前）。
- 顶部展示当前 baseline 位置；支持查看某回合对应的对话与 diff。

**关键交互**：回退为破坏性操作 → 二次确认 + 说明影响范围；节点点击可展开看该回合对话/diff。

**Review**
- ✅ `color-not-only`：状态点用颜色 + 文字/图标（待接受/已接受/已回退）。
- ✅ `confirmation-dialogs` + `destructive-emphasis`：回退需确认、用警示色，与查看操作分离。
- ✅ `timeline-clarity`：清晰的时间线连线 + 里程碑标记区分「已接受基线」。
- ⚠ 已接受里程碑要明确表达「不可再回退到此之前」，避免用户误以为能无限回滚。
- ⚠ 长历史需 `virtualize-lists`；对话永久留存与代码快照已清理的差异需在 UI 讲清楚。

## 全局 Review（跨页面，对照 ui-ux-pro-max 关键项）
| 优先级 | 项 | 结论 |
|---|---|---|
| CRITICAL | 无障碍对比度 4.5:1 | 暗色下正文用 `#F8FAFC`，次要 `#94A3B8`，达标 |
| CRITICAL | 触控 ≥44pt、间距 ≥8px | 输入/导航/审批按钮均满足 |
| HIGH | 图标用 SVG 非 emoji | 统一 Lucide 线性图标 |
| HIGH | 底部导航 ≤5、图标+文字、高亮当前 | 满足 |
| HIGH | 安全区适配（刘海/手势条） | 头部与底部栏预留 padding |
| MEDIUM | 动效 150–300ms、尊重 reduced-motion | 运行呼吸点/卡片进入统一节奏 |
| MEDIUM | 表单/反馈：错误就近、可恢复 | 连接失败、审批、提交均有反馈 |

## 8 个视觉版本（详见 mockups/index.html）
| 版本 | 名称 | 气质 | 主色/效果 | 适用倾向 |
|---|---|---|---|---|
| V1 | Terminal Green (OLED) | 极客、纯正开发者 | 深黑 + 终端绿 `#22C55E`，等宽点缀 | **推荐默认**，最贴工具属性 |
| V2 | Aurora Glass | 高级、现代、通透 | 深蓝渐变 + 毛玻璃卡片 + 靛紫 | 想要「精致 SaaS」质感 |
| V3 | Cyberpunk HUD | 潮酷、赛博 | 近黑 + 霓虹青/品红 + 扫描线 | 强调个性、demo 吸睛 |
| V4 | Claude Warm | 温暖、友好、亲和 | 暖夜色 + 陶土橙 `#D97757` | 弱化机器感、更亲人 |
| V5 | Linear Minimal | 克制、锐利、高密度 | 近黑 + 靛蓝 `#5E6AD2`、细线 | 效率党、信息密集 |
| **V6** | **Swiss Light (Paper)** | 克制、专业、网格化 | **唯一浅色**：纸白 + 墨黑 + 单一电光蓝 `#2563EB`、极小圆角 | 日间/阅读向，Swiss Modernism 风 |
| **V7** | **Neo Brutalism** | 张扬、潮玩、Gen-Z | 奶油底 + 3px 纯黑粗边 + 硬阴影(无模糊) + 番茄红 `#EF3E36` + Space Grotesk 粗标题 | 个性化、demo 极吸睛 |
| **V8** | **Synthwave Sunset** | 复古、赛博、氛围 | 深紫渐变 + 品红霓虹 `#FF4D9D` + 透视网格 + 辉光 | 与 Cyberpunk 冷青形成暖调对照 |

> 新增 V6/V7/V8 依据 ui-ux-pro-max：V6 取 `Swiss Modernism 2.0`（严格网格 + 单一强调 + 高对比），并补齐设计系统缺失的**浅色场景**（前 5 版全为暗色）；V7 取 `Neo Brutalism (Mobile)`（粗黑边 + 硬偏移阴影 + 高饱和色块）；V8 为暗色蒸汽波，区别于 V3 的冷色 HUD。所有版本仍由 CSS 变量驱动，页面模板零改动即全部生效。
